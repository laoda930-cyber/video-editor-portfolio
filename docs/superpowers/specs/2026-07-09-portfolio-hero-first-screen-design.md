# 作品集首页第一屏设计规格

## 目标与范围

为视频剪辑师李欣霞的作品集新增一个完整占据首个视口的 Hero。首屏应先传达现代、克制、专业的高级感，再通过暖色细节为后续小猫互动保留温度与叙事入口。

本轮交付为独立文件 `hero-first-screen.html`，包含首屏初始状态、五组动态占位画面及“下一个”按钮触发的竖向翻页。所有样式和脚本内嵌在该文件中，可直接用浏览器打开，不依赖现有 React 应用。本轮不修改现有 `index.html` 或 `src` 目录，不生成、不加载、不显示小猫素材，也不实现小猫状态机、长按僵持、按钮飞走或尾巴彩蛋。

## 信息结构

### 顶部导航

- 左侧显示“李欣霞 / VIDEO EDITOR”品牌标识。
- 右侧保留“作品”“关于”“联系”三个入口。
- “作品”聚焦首屏平板预览；“关于”聚焦首屏的个人定位文字；“联系”使用现有邮箱链接。
- 导航在桌面端轻量悬浮于首屏顶部，移动端压缩间距但不隐藏核心联系入口。

### 左侧文案

- 小标签：`VIDEO EDITOR · 2026`。
- 主标题：`让每一帧，都值得停留`。
- 定位文案：`专注商业短视频，用剪辑驱动传播与转化。`
- 能力标签：`品牌宣传`、`产品推广`、`营销内容`。
- 不展示未经确认的播放量、客户数或爆款数量。

### 右侧作品设备

- 使用竖屏平板模型承载作品预览，设备有深空灰细边框、圆角屏幕、轻微玻璃反射和柔和弥散阴影。
- 平板显示当前画面的序号、标题和类别；这些信息与画面一起翻页。
- “下一个”按钮位于平板右下侧，采用深色半透明毛玻璃材质和向下箭头。
- 纸箱结构位于按钮后方，只露出极淡的边缘，不构成当前首屏的视觉焦点。

## 五组动态占位画面

动态占位画面全部用 HTML 与 CSS 构成，避免引入临时图片或远程视频依赖。每组画面拥有独立色彩、图形语言和缓慢循环的内部动效：

1. 品牌视觉叙事：暖杏与深棕，使用裁切字幕和缓慢移动的光带。
2. 产品质感短片：银灰与冰蓝，使用玻璃产品轮廓和环形高光。
3. 商业增长内容：鼠尾草绿与炭黑，使用层叠数据块和节奏线。
4. 情绪剧情短片：暮紫与珊瑚橙，使用电影画幅和柔焦光斑。
5. 知识动画合成：奶油白与克制蓝，使用网格、路径和模块化图形。

画面动效应保持抽象，不冒充真实作品。未来接入真实竖版视频时，只替换每页的视觉内容层，不改变设备、翻页或按钮结构。

## 视觉系统

- 页面背景：暖灰白 `#F5F3F0`。
- 主文字：近黑 `#1A1A1A`；说明文字使用对比度足够的中灰。
- 点缀色：浅杏橙 `#E8A87C`，仅用于当前序号、微光或小面积强调。
- 字体：使用本地系统无衬线字体栈，标题采用 Light/Regular 视觉重量，不依赖网络字体。
- 阴影：平板和按钮统一使用低透明度、长模糊半径的柔和阴影。
- 布局：桌面端使用 12 列感知栅格，左右安全边距约 80px；左侧文案占 4–5 列，右侧设备占 6–7 列。
- Hero 高度至少为 `100svh`，独立文件不包含首屏以下的作品列表。

## 交互与动画

### 翻页逻辑

- 点击“下一个”后，当前页向上移动至平板屏幕外；下一页从屏幕底部同步进入。
- 整页包括背景、标题、序号和装饰图形，以统一容器完成位移，形成类似抖音的竖向整屏吸附感。
- 翻页时长约 420ms，使用先快后稳的缓动曲线。
- 第五页继续点击后循环回第一组。
- 动画过程中暂时忽略重复点击，防止索引错乱和画面重叠；结束后立即恢复按钮。

### 初始微交互

- “下一个”按钮使用约 3 秒周期的克制呼吸光效。
- 悬停时按钮轻微上浮，按下时下沉约 2px，键盘聚焦时显示清晰焦点环。
- 本轮只通过按钮触发翻页。滚轮、触摸拖动、长按和角色触发所需的接口位置保留，但不提供未完成的交互暗示。

### 后续小猫结构预留

- Hero 内部保留独立的 `scene`、`device`、`control`、`box` 和 `character-layer` 层级边界。
- 交互状态由当前画面索引与是否正在切换组成；未来可在此基础上增加点击计数、停留时间和角色阶段，而无需改写画面轮播。
- `character-layer` 本轮为空且不请求任何素材，纸箱仅为静态、低可见度结构。

## 响应式行为

- 桌面端：左右双栏，平板右置，按钮位于设备右下侧。
- 平板端：缩小左右间距和设备尺寸，仍维持双栏关系。
- 移动端：文案在上、平板在下；设备保持竖屏比例，按钮移到设备下方或右下安全区，避免横向溢出。
- 所有可点击目标至少 48px，页面不禁用缩放，不产生横向滚动。

## 可访问性与降级

- “下一个”使用原生 `button`，名称包含当前进度，例如“下一个作品，当前第 1 个，共 5 个”。
- 动态装饰图形对屏幕阅读器隐藏；作品标题和类别保持可读取。
- 使用 `aria-live="polite"` 汇报切换后的作品标题，但不朗读装饰内容。
- 遵循 `prefers-reduced-motion`：取消内部循环动效，将翻页改为短淡入淡出或即时切换。
- 焦点顺序与视觉顺序一致，并保留高对比度焦点样式。

## 单文件结构与数据边界

- `hero-first-screen.html` 包含语义化 HTML、一个内嵌 `<style>` 和一个内嵌 `<script>`，不引入框架、构建工具或外部 CDN。
- HTML 按 `hero`、`device`、`scene-track`、`controls`、`box` 和 `character-layer` 划分清晰区域。
- 五组场景以静态 HTML 项呈现，标题、类别和序号通过 `data-*` 属性提供给脚本读取。
- 脚本只负责当前索引、竖向位移、切换锁和可访问性播报；视觉样式与内部循环动效留在 CSS。

数据流保持单向：按钮请求下一页，脚本更新画面轨道与当前索引，动画结束后释放切换锁。页面不发起网络请求，因此不需要加载或网络错误状态。

## 验证标准

- 首个视口完整呈现导航、左侧文案、竖屏平板、按钮和弱化纸箱，不露出下方作品卡片。
- 存在五组名称、色彩和构图均可区分的动态占位画面。
- 连续点击按钮可按顺序浏览五组画面并正确循环，切换方向始终为旧页上移、新页从下进入。
- 快速重复点击不会跳过索引、重叠画面或锁死按钮。
- 桌面、平板和手机宽度下无横向溢出，按钮可通过鼠标、触摸和键盘使用。
- 开启减少动态效果后，无持续循环动画，内容和切换功能仍完整可用。
- 独立文件可通过 `file://` 直接打开，五组画面与按钮均正常工作。
- 现有 `index.html`、React 组件、样式、测试和构建入口均未被修改。

## 非目标

- 不接入真实视频、远程图片、音效或网络字体。
- 不实现小猫素材、角色动画及相关隐藏彩蛋。
- 不包含首屏以下的作品分类、弹窗或页脚。
- 不修改、嵌入或替换现有网站页面。
