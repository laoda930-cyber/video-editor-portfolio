# 四层视差作品集改版设计

## 目标

将现有浅色作品画廊重做为参考图中的梦幻天空与花海作品集。页面保留四个清晰入口：`首页`、`项目介绍`、`作品集`、`关于我`，作品图片和视频先使用占位内容。

## 信息结构

1. 首页：整屏英文标题、两行中文说明、`开始 start` 与 `关于我 about me` 两个按钮。
2. 项目介绍：左侧使用“我把想法变成可以被看见、被体验的内容。”大标题和一段定位说明，右侧展示三张视频占位卡。
3. 作品集：展示六张精选视频占位卡，点击后提示“视频待上传”，不打开空播放器。
4. 关于我：只保留视频剪辑师定位、邮箱与电话，不加入履历、项目数据或个人信息栏目。

## 视觉系统

- PC 版内容宽度最大约 1700px。
- 主色为浅蓝天空、白色云雾、绿色花海和少量紫粉花朵；正文使用深灰黑，强调文字使用纯蓝色。
- 顶部导航固定居中，当前页签为白色胶囊，其他页签保持透明。
- 作品卡采用参考图中的方形/近方形画面、圆形播放按钮、标题与简短副标题，卡片圆角不超过 16px。
- 参考截图只作为样式依据，不直接当背景图，避免把截图中的导航和文字重复渲染进页面。

## 四层视差

页面视觉由四个独立层组成，均不拦截鼠标事件：

1. `background`：固定天空渐变，位移速度为 0。
2. `subject`：主视觉云环和光区，按滚动距离轻微反向移动。
3. `midground`：远处花海，移动幅度高于主体。
4. `foreground`：近景花草，跟随滑动并拥有最大位移，形成明确景深。

滚动值由一个被 `requestAnimationFrame` 节流的监听器写入根节点 CSS 变量。系统启用“减少动态效果”时，全部图层位移归零。移动端降低图层位移并简化布局。

## 可替换素材接口

四层都保留独立类名和 `data-layer` 属性。后续拿到透明 PNG 后，可分别替换对应层的 `background-image`，无需调整 React 结构和滚动脚本。

## 验收标准

- 页面能通过 Vite 构建并在 GitHub Pages 子路径中运行。
- 首屏、项目介绍、作品集、关于我四段均可通过固定导航到达。
- DOM 中存在四个独立图层，前景具有最大滚动速率，背景固定。
- 滚动后根节点 CSS 变量更新，且存在 `prefers-reduced-motion` 降级。
- 所有占位视频点击后只显示待上传提示。
- 1280px 以上桌面宽度与窄屏布局均无横向溢出。
